Part 1: LP: The Simplex Method

What It Solves

Simplex solves linear programming problems: maximize or minimize a linear objective under linear constraints. In this tool, the objective is z = c1x1 + c2x2 with a geometric feasible region and a matching tableau view.

How It Works

It transforms constraints into standard form with slack/surplus variables, starts from a basic feasible solution, then repeatedly pivots. Each pivot chooses an entering variable (improves objective) and leaving variable (keeps feasibility) until no improving reduced cost remains.

When To Use It

Use Simplex for planning and allocation tasks such as production, logistics, and scheduling when relationships are linear. It is also valuable for teaching optimization behavior, degeneracy, infeasibility, and unboundedness through explicit step-by-step pivots.

Educational Content & Learning Guide
Key Insight: Optimum At A Vertex
The feasible set is a convex polygon in 2D. If an optimum exists, at least one optimal solution is a vertex of that polygon, so moving vertex-to-vertex is sufficient.
Problem Formulation (Real Scenario)

Scenario: A factory produces two products, A and B. Each unit uses limited resources (machines, labor, material). The goal is to maximize profit.

Inputs: c1 and c2 are profit per unit of A and B. Each constraint row is one resource limit (e.g., machine hours).

Decision variables: x1 = units of product A, x2 = units of product B.

Output: The optimal mix (x1, x2) and the maximum profit z*.

Algorithm Selection: Why LP + Simplex

Why LP: All relationships are linear (profits and resource usage scale linearly with production).

Why Simplex: It navigates vertex-to-vertex on the feasible polygon and guarantees optimality when reduced costs are nonnegative.

When it does not apply: Nonlinear constraints/objectives, nonconvex regions, or integer-only requirements without relaxation.

Execution & Correctness (Why It Works)
The feasible region is convex, and any optimum occurs at a vertex. Simplex pivots along edges to improve the objective. When all reduced costs are nonnegative, no adjacent vertex improves the objective, so the current solution is optimal. Dual feasibility and a near-zero duality gap confirm correctness.
Decision Trace: Entering/Leaving Rules

Entering variable: Choose the most negative reduced cost (Largest Coefficient rule) or the smallest index with negative reduced cost (Bland's rule).

Leaving variable: Use the minimum ratio test b_i / a_ij for rows with a_ij > 0 to keep feasibility.

These decisions are shown step-by-step in the explanation panel and the highlighted tableau.

Part 2: Strassen's Fast Matrix Multiplication

What It Solves

Strassen computes matrix multiplication faster than classical O(n^3) by reducing the number of block multiplications. It targets dense square matrices and demonstrates algorithmic speedup through algebraic restructuring.

How It Works

Each matrix is partitioned into four quadrants. Instead of 8 quadrant multiplications, Strassen derives 7 products (P1..P7) and combines them to obtain C11..C22. The same idea applies recursively to submatrices until reaching a base case.

When To Use It

It is most useful for larger matrix sizes where multiplication cost dominates addition overhead. Practical implementations usually switch to classical multiplication below a cutoff size for better cache behavior and numerical stability control.

Part 3: Diffie-Hellman Key Exchange

What It Solves

Diffie-Hellman Key Exchange is a protocol that lets two parties derive the same shared secret over an insecure channel without transmitting that secret directly.

How It Works

Both parties agree on public parameters (p, g), choose private exponents, publish g^a mod p and g^b mod p, then compute (g^b)^a mod p and (g^a)^b mod p. Modular exponentiation makes both results match as the same shared key.

When To Use It

Use it when establishing a session key across a network (for example in TLS key agreement variants). DH provides secrecy of the shared key, but must be combined with authentication to prevent man-in-the-middle attacks.

Educational Content & Learning Guide
Key Insight: Discrete Logarithm Hardness
Eve can observe g, p, A, and B, but recovering a from A = g^a mod p is computationally hard for large primes.
Worked Example (Preset 1)
p=23, g=5, a=6, b=15 A = 5^6 mod 23 = 8 B = 5^15 mod 23 = 19 Alice computes S = 19^6 mod 23 = 2 Bob computes S = 8^15 mod 23 = 2 Both obtain shared secret S = 2.
